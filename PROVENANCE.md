# PROVENANCE — Vexor's upstream reference ledger

**What this is:** the single, authoritative map of every place Vexor's code
reimplements, ports, or byte-exact-matches behavior from an upstream Solana
client (Agave, Firedancer, Sig). It replaces scattered inline `// Agave x.rs:NNN`
comments with one maintained document, so the codebase stays clean **and** the
parity trail stays findable for maintenance.

**Why it exists:** Vexor targets byte-for-byte Agave compatibility. When Agave
changes, a maintainer needs to find which Vexor code tracks the changed behavior.
This ledger is that index — and it doubles as the attribution required for
reimplementing Apache-2.0 code (see `NOTICE`).

---

## How entries are keyed (rot-resistant)

Entries are keyed by a **stable provenance ID** and by **file + function**, never
by a raw line number (line numbers drift on every edit; IDs and function names do
not). In the code, a one-line anchor marks the spot:

```zig
// @prov:cost-model.data-cost
```

and this ledger holds the full detail for that ID. Grep works both directions:
`grep -rn '@prov:cost-model.data-cost' src` → the code; the table below → the
upstream source.

**Kind** column: `reimpl` (behavior reimplemented in Zig from reading upstream),
`port` (logic translated near-line-by-line), `byte-exact` (verified bit-identical
against upstream output / KAT), `spec` (implemented from a published SIMD/spec,
upstream used only as a cross-check).
